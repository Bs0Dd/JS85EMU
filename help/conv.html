BASIC data converters (in the data loading and saving section)


    The allocated variables and programs are stored in the microcomputer's RAM in a proprietary format. Therefore, BASIC data converters are used to obtain this data in a readable text format and load it back.
    The emulator has three converters:


Allocated variables converter


example contents of the file "mk85_vars.txt"

    This converter allows you to get a list of all allocated variables (by default there are 26 of them (from A to Z) plus a special text variable $), whose values ‚Äã‚Äãare not equal to zero. When you click on the "Get vars list" button, the file "mk85_vars.txt" is saved to disk, an example of which is seen above. If the number of allocated variables is more than 26, the variables following Z, according to the documentation, are referred to as Z(number).
    To display specific characters from the MK 85 code page, analogous or similar in style Unicode characters are used, so some of them may not display correctly in your text editor.
    A table of relation between Unicode and MK 85 characters is presented at the bottom of the page.


BASIC programs to text code converter


example contents of the file "mk85_programs.txt"

    This converter is designed to receive programs stored in RAM in the form of text code (a.k.a. listing). The "Get programs" button saves the file "mk85_programs.txt" to disk, an example of which is also given above. The ability to save listings allows you to edit programs using a regular text editor, as well as simply publish them for others to load into an emulator or retype onto a real device. Before the beginning of the code of each program, its number (0-9) in the memory of the virtual microcomputer is indicated. Unicode is used to display specific characters, however, for ease of editing, some characters (only outside the quotes) are replaced with equivalents, for example '‚Üë' with '^', '‚â§' with '<=', '‚â•' with '>=', 'œÄ' with 'PI', '‚â†' with '!=', and only in numbers '·µâ' with 'E' and '·µâ‚Åª' with 'E-'.


Text BASIC code into programs converter


example of the contents of a file with programs

    Since programs can be downloaded from memory, it is logical to have a vice versa thing for loading text code (listing) into a virtual microcomputer. For this purpose, there is a "Load program from file" button and a corresponding file selection field.
    The file can contain from 1 to 10 programs, separated from each other by two blank lines. Before starting each program, you can specify its number in the format "Program *", where * is a number from 0 to 9 (if the cell is occupied, the conversion will be interrupted). Otherwise, the program will be loaded into the first free cell.
    Lines that do not begin with their number are considered as comments and are ignored during conversion. Commands and variables can be entered in any case (but during conversion they will be converted to standard microcomputer upper case, i.e. "print a$;" => "PRINT A$;"). For convenience, some characters (only outside quotes) can be replaced with equivalents, for example '‚Üë' with '^', '‚â§' with '<=' or '=<', '‚â•' with '>=' or '=>', 'œÄ' with 'PI', '‚â†' with '!=' or '><' or '<>', and only in numbers '·µâ' with 'E' and '·µâ‚Åª' with 'E-'.
    If the conversion of all programs is completed successfully, a notification will appear with the number of processed programs and the numbers of the cells in which they were saved. If an error occurs, the conversion will be interrupted and the notification will indicate the reason for the error (along with the line number in the file), as well as the number of processed before this programs and their cell numbers.

    Characters marked in red (1 to 10) cannot be entered using the microcomputer keyboard, so the "CHR dec. code" command should be used instead, for example "CHR 10" for the character '‚Ä¶'.
    The 0 character does not exist because 0 is the end-of-line character in a microcomputer. Character 96 (0x60) does not have its own style, since it is specified by the user using the "LETC code or variable" command, for example ' LETC "4ERE4E4" ' or ' LETC G$ '. The "CHR 96" command should always be used for this symbol.
